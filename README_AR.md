# نظام نقطة الشحن وبيع العصير — نسخة v8

هذه النسخة تعمل بنفس مزامنة كاش توب عبر REST API، مع حفظ محلي فوري وطابور مزامنة تلقائي.

## الاتصال والمزامنة

- API: `https://cash-top-api-2026.vercel.app/api/rtdb`
- الشركة: `COMPANY001`
- الفرع: `main`
- مسار التطبيق: `companies/COMPANY001/branchesData/main/data/juicePOS`
- كل عملية تُحفظ محلياً أولاً ثم تُزامن في الخلفية.
- المستخدمون والصلاحيات محفوظون في مجموعة `users` داخل مسار التطبيق نفسه ويتزامنون بين الأجهزة.

## تسجيل الدخول

- المدير: اسم المستخدم `admin`
- كلمة المرور الافتراضية للمدير: `78789852`
- المستخدمون الإضافيون يتم إنشاؤهم من قسم «المستخدمون والصلاحيات».
- كلمة مرور كل مستخدم تُحفظ كبصمة Hash ولا تُخزن كنص ظاهر.

## الصلاحيات

يمكن تحديد صلاحيات دقيقة لكل مستخدم، منها:

- عرض الرئيسية والعمليات، وإنشاء العمليات، وتعديلها أو حذفها، وتنزيل الفواتير.
- عرض المخزون أو إدارته.
- عرض الزبائن، إدارتهم، السداد والتسويات، وتنزيل كشوف الحساب.
- عرض التقارير أو تنزيلها.
- عرض المصاريف أو إدارتها.
- عرض الإعدادات أو تعديلها.
- إدارة المستخدمين والصلاحيات.

القوائم والأزرار غير المسموح بها تُخفى من واجهة المستخدم، ويوجد فحص صلاحية داخل وظائف الحفظ والتعديل والحذف أيضاً.

## تحسينات شاشة العملية

- عند كتابة أول حرف أو أكثر من اسم الزبون تظهر فوراً الأسماء التي تبدأ بنفس الأحرف.
- خيار «شراء عصير» لم يعد محدداً تلقائياً عند فتح عملية جديدة.
- مربع كمية العصير أصبح أكبر وأوضح مع أزرار + و− كبيرة.

## PWA والكاش

- إصدار الكاش: `juice-pos-v8-users-permissions`.
- التطبيق يستخدم Cache First للملفات والواجهة لفتح سريع حتى بوجود الإنترنت، ثم يحدث الكاش في الخلفية.
- طلبات قاعدة البيانات لا تدخل كاش Service Worker حتى تبقى المزامنة حديثة.
- للتثبيت من Chrome يجب تشغيل الملفات على HTTPS أو localhost.
