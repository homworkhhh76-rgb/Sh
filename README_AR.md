# نظام نقطة الشحن وبيع العصير — نسخة مزامنة كاش توب v7

هذه النسخة أصلحت المزامنة بإزالة اعتماد التطبيق على Firestore بالكامل. التطبيق يستخدم الآن نفس نمط قاعدة البيانات الذي نجح في برنامج كاش توب:

- API: `https://cash-top-api-2026.vercel.app/api/rtdb`
- الشركة: `COMPANY001`
- الفرع: `main`
- مسار هذا التطبيق: `companies/COMPANY001/branchesData/main/data/juicePOS`

## طريقة العمل

1. كل زر يحفظ أولاً في `localStorage` فوراً، لذلك لا يعلق الحفظ بانتظار الإنترنت.
2. كل عملية تدخل طابور مزامنة محلي دائم.
3. المزامنة ترفع السجلات عبر `GET / PUT / DELETE` إلى API كاش توب.
4. عند فتح التطبيق، وعند عودة الإنترنت، وكل 12 ثانية، يتم تنزيل آخر البيانات ثم رفع العمليات المعلقة.
5. إذا فشل الاتصال، تتحول الحالة من «جاري الاتصال» إلى رسالة خطأ واضحة ولا تبقى الدائرة تلف بلا نهاية.
6. كلمة المرور الافتراضية هي `78789852`، وتغييرها من الإعدادات يحفظ البصمة في `settings/security`.

## PWA والكاش

- إصدار الكاش: `juice-pos-v7-cashtop-rest`.
- الصفحات والملفات المحلية تعمل Cache First وتفتح من الكاش فوراً، ثم يتم تحديثها في الخلفية.
- طلبات قاعدة البيانات مستثناة من Service Worker cache دائماً حتى لا تظهر بيانات قديمة.
- يجب تشغيل التطبيق من HTTPS أو localhost حتى يعمل Service Worker وخيار التثبيت في Chrome.
