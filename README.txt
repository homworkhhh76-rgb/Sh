كاش توب — إصدار Firebase + Offline (الفواتير وملفات الزبائن)

الملفات كلها في مستوى واحد بدون مجلدات.

المزايا الرئيسية:
- يعمل محلياً بدون إنترنت، وكل تغيير يبقى في قائمة المزامنة حتى نجاح رفعه إلى Firebase.
- ملف مستقل لكل زبون يعرض الشحن والمشتريات والديون والدفعات، مع تقرير قابل للطباعة أو الحفظ PDF.
- فاتورة موحدة: اختيار الزبون أولاً، ثم إضافة شحن جوال وصنف/عصير واحد أو أكثر في معاملة واحدة.
- طرق الدفع: نقدي، تحويل بنكي، Jawwal Pay، أو دين، مع خانة ملاحظات لاسم صاحب الحوالة وغيرها.
- قسم مستقل للمصاريف، ويظهر سجل المصاريف وإجماليها داخل التقارير.
- تعديل وحذف العمليات والمصاريف متاحان للمدير فقط، مع عكس أثر العملية على رصيد الزبون والمخزون.
- المحافظة على العمليات القديمة المنفصلة والبيانات السابقة دون تغيير مسار Firebase.
- مزامنة الزبائن، الأرصدة، العمليات، الفواتير، المصاريف، المخزون، الأجهزة، المستخدمين وPIN.

بيانات الدخول الافتراضية:
- المدير: PIN 0000
- مستخدم 1: PIN 1111
- مستخدم 2: PIN 2222
يُنصح بتغيير كلمات المرور من الإعدادات بعد التشغيل.

إعداد Firebase:
1) فعّل Realtime Database في مشروع Firebase.
2) فعّل Anonymous Authentication من Authentication > Sign-in method.
3) انشر firebase-rules.json أو نفّذ:
   firebase deploy --only database
4) للنشر على Firebase Hosting:
   firebase deploy --only hosting

ملف إعداد المشروع:
firebase-config.js

مسار بيانات التطبيق (لم يتغير حفاظاً على البيانات الحالية):
cashTopChargeApp/v3/state
